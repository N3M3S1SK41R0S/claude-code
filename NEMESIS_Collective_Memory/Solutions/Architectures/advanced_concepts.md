### 14. SEMANTIC VERSIONING FOR DECISIONS
```yaml
concept: DECISION_VERSIONING
status: proposed
description: |
  Versionne tes décisions architecturales et de code
  avec historique, rationale, et possibilité de rollback.

version_structure:
  decision_id: unique identifier
  version: semver (1.0.0)
  timestamp: when decided
  context: why this decision
  rationale: reasoning
  alternatives: what was rejected
  consequences: expected outcomes
  status: active | superseded | deprecated

operations:
  create: new decision
  update: modify existing (version bump)
  supersede: replace with new decision
  deprecate: mark as outdated
  rollback: revert to previous version

queries:
  - "Show decision history for authentication"
  - "Why did we choose PostgreSQL over MongoDB?"
  - "What changed between v1 and v2 of the API design?"
  - "Rollback database decision to v1.2.0"
```
